## Milestone v1.1 Requirements

### Input Mapping
- [ ] **INPUT-01**: The system translates raw FTC Gamepad input (analog sticks, buttons) into immutable human-intent `RobotAction`s.
- [ ] **INPUT-02**: Stick input is processed with configurable deadbands and exponential curves before being dispatched to the state store.

### Hardware Odometry & Forward Kinematics
- [ ] **ODOM-01**: An IO layer (`PinpointIO` or similar) reads raw `(X, Y, Heading)` pose data from I2C hardware sensors (e.g. goBILDA Pinpoint / OTOS) and dispatches it as an immutable state update without running math on the main CPU.
- [ ] **ODOM-02**: A software-based forward kinematics engine calculates estimated pose from raw wheel encoder changes for systems without hardware odometry.

### Field-Centric Driving
- [ ] **DRIVE-01**: The pure holonomic kinematics engine accepts the current odometry heading to transform human joystick inputs (X/Y) into field-centric drive speeds.

### FTC Dashboard Integration
- [ ] **DASH-01**: The pure `RobotState` and calculated pose are serialized and pushed to FTC Dashboard telemetry for real-time debugging.
- [ ] **DASH-02**: Field-centric pose and module vectors are drawn on the FTC Dashboard field canvas.
