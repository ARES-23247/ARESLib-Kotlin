## Validated Requirements

### Category: Advanced Controller Architecture
- [x] **CTRL-01**: The system must provide a unified `Controller` interface that wraps raw hardware inputs (FTC Gamepad or FRC GenericHID).
- [x] **CTRL-02**: The controller must support button state tracking (isPressed, isHeld, isReleased).
- [x] **CTRL-03**: The controller must support edge-detection event triggers (onTrue, onFalse) for seamless command/action binding.
- [x] **CTRL-04**: Analog triggers must be configurable with thresholds to act as digital buttons.

### Category: NetworkTables 4 (NT4) Server
- [x] **NT-01**: A pure Kotlin, Android-compatible NT4 server (or WebSocket implementation) must be built or integrated to run on the FTC Control Hub.
- [x] **NT-02**: The NT4 server must publish robot state (Pose2d, velocities, hardware states) to standard NT4 paths recognizable by AdvantageScope.
- [x] **NT-03**: The NT4 server must subscribe to tuning variables (e.g., PID gains) to allow live adjustments from FRC Dashboards (like Elastic or Shuffleboard).

### Category: Hardware Integration & Vision
- [x] **HW-01**: Create physical vision wrappers for Limelight 3A and FTC VisionPortal (AprilTags) that implement the `VisionIO` abstraction.
- [x] **HW-02**: Implement a centralized `RobotConfig` or `RobotMap` pattern to map physical hardware ports to abstract IO interfaces cleanly.

### Category: Data Logging
- [x] **LOG-01**: Implement an on-device data logger that writes telemetry to the Control Hub's local storage (SD Card) in either CSV or WPILog format for post-match analysis.
