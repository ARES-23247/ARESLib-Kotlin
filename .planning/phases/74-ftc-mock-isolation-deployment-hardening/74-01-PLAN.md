# Phase 74 Plan 01: FTC Mock Isolation & Deployment Hardening

## Objective

- **What**: Resolve the critical bugs identified during the pre-hardware audit. This includes:
  1. Creating proper universal IMU, RevHubOrientationOnRobot, and navigation mocks in `:ftc-mocks`.
  2. Migrating the deprecated legacy `BNO055IMU` calls in `FtcRevHubIO` and testing OpModes to the standard universal `IMU` interface.
  3. Inserting the missing `waitForStart()` call in `ARESLibOpMode` to fix immediate startup lifecycle crashes.
  4. Wiring up the unhandled `JoystickDriveIntent` in `DriveReducer` to successfully translate joysticks to motor speeds.
  5. Refactoring active motor encoder resets to use zero-latency **Software Offsets** rather than blocking physical I2C `STOP_AND_RESET_ENCODER` calls.
- **Why**: These changes ensure the robot does not crash or experience severe latency spikes when deployed to real hardware, while preserving local desktop simulation capabilities.
